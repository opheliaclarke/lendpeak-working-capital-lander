TEAM GUIDE · GOOGLE ADS CONVERSIONS

🎯 Create the Conversion Actions

Exact click-by-click setup for account GCALIT x LendPeak (720-674-6191). The screens below are annotated mockups of the real Google Ads flow — type the highlighted values exactly. Do this once; the naming is built so future lead uploads "just work."

The most-efficient setup = just 2 conversion actions. Lead (from the website) to optimize on at launch, and Funded Deal (uploaded from GoHighLevel) which becomes the money goal later. One Google Sheet feeds all future uploads. Don't create duplicates — extra actions split your data and slow learning.

⭐ Naming convention — read this first (it's the whole trick)

When you upload leads/deals later, Google matches each row to a conversion action by its name, character-for-character. So we lock simple, exact names now and never rename them:

Conversion action nameCategoryHow it's recordedValueRole
LeadSubmit lead formWebsite (Google tag)No valuePrimary at launch
Funded DealConverted leadImport (uses GCLID)Use different values (USD)Secondary → Primary later
Qualified Lead (optional)Qualified leadImport (uses GCLID)No valueSecondary signal
Rules that prevent broken uploads: names are case-sensitive and must match the upload sheet's "Conversion Name" exactly (Funded Deal ≠ funded deal ≠ Funded  Deal). No trailing spaces. Never rename an action after go-live, and never delete + recreate (make a new name instead).

Step C2 — The Sheet format (give this to the GHL team)

In the template, set the timezone to match the account, then GHL appends one row per funded deal:

Row 1 (parameters):  Parameters:TimeZone=America/New_York

Header row + columns GHL fills:
Google Click ID      -> {{contact.gclid}}
Conversion Name      -> Funded Deal          (must match the action name EXACTLY)
Conversion Time      -> 2026-06-26 15:30:00   (yyyy-MM-dd HH:mm:ss, Eastern)
Conversion Value     -> 25000                 (the funded amount, number only)
Conversion Currency  -> USD
For a Qualified Lead upload later, GHL writes the same row but with Conversion Name = Qualified Lead. Same Sheet, same pipe — that's why the naming had to be locked first.

Part C2 · ValueTrack — Final URL suffix (marketing/our side, in Google Ads)

So Leo's team can save which campaign / ad group / keyword / device drove each lead, we append ValueTrack values to every click URL. (gclid is added automatically by auto-tagging — confirm Account settings → Auto-tagging = ON.) Set this once:

Campaign LP | MCA | Search | US → Settings → Additional settings → Campaign URL options → Final URL suffix, paste:

utm_source=google&utm_medium=cpc&campaignid={campaignid}&adgroupid={adgroupid}&keyword={keyword}&matchtype={matchtype}&device={device}&network={network}&creative={creative}
Google swaps each {…} for the real value at click time, e.g. ?gclid=Cj0K…&campaignid=2231&keyword=merchant%20cash%20advance&matchtype=e&device=m. Leo's team's snippet (their setup page) reads these exact names into the lead form. The field names here must match their hidden-field names character-for-character — they do. Click Test in the URL-options box to confirm the page still loads with the suffix.

Part D · Point the campaign at the right goal

  1. Goals → Conversions → Summary: confirm Lead = Primary, Funded Deal = Secondary.
  2. In the campaign → Settings → Conversion goals → use account-default (so it optimizes to Lead now).
  3. Once Funded Deal has ~15–30 imports, flip it to Primary and bid to it (Target CPA / Max Conversion Value). That's when Google starts optimizing to real money.

✅ Verify before unpausing

  1. Submit a test lead with ?gclid=TEST123 → the Lead action shows "Recording conversions" in Tag Assistant.
  2. Add a test row to the Sheet (gclid TEST123, name Funded Deal) → next day it imports with no errors.
  3. Delete test data → unpause and scale.
